# CCSwitchMulti v3.16.5-6

本版集中修复 Codex MultiRouter 的 Provider 选择边界、模型目录重复、刷新生命周期、GPT-5.6 选择器元数据，以及新模型的用量与计费。

## MultiRouter 修复

- 设置向导的 Provider 选择改为独立的本次草稿选择。现在可以取消某个 Provider，且不会删除该 Provider，也不会因页面重新渲染自动重新选中。
- 正常路由设置页只刷新当前方案中已启用 route 引用的 Provider。未勾选或已停用的 Provider 不再请求模型列表、写回目录或重新进入聚合 catalog。
- route 停用后立即清除该 Provider 的旧刷新状态，并在保存请求发出前让尚未完成的模型请求失效，避免保存等待窗口内的迟到结果按旧路由写回 catalog；重新启用后会重新读取模型，不会被旧 attempt key 永久拦截。
- 聚合模型目录、Provider 变更同步和 Codex picker 注入都排除停用 route，修复 `5.4`、`5.5` 等模型重复出现的问题。
- 兼容旧版缺少 `targetProviderId` 的 inline route：仅通过 route 名称和模型语义定位对应 Provider，完成一次迁移，不恢复“刷新全部 Provider”的旧行为。
- 官方 Codex 模型缓存现在只为同 slug 的已路由模型补充元数据，不再把未路由的官方独有模型追加回 MultiRouter。

## GPT-5.6 与 Luna

- GPT-5.6 Sol、Terra 保留官方 Codex 的展示名、`max` / `ultra` 推理档位、默认推理强度和 `fast` 速度档，不再被通用 GPT 模板降级成旧版四档。
- HTTP-only MultiRouter 不再把官方标记为 `prefer_websockets` 的模型写进运行时目录，避免把 `gpt-5.6-luna` 暴露给不支持 Responses WebSocket 的本地代理。
- Provider 内联模型同时投影官方推理元数据的 snake_case 与 camelCase 字段，保持 Codex Desktop 不同读取路径的一致性。

## 用量与定价

- Codex 流式 usage 解析兼容 `response.done` 以及事件顶层的 `usage`，修复部分 GPT-5.6 官方 OAuth Responses 流成功返回却统计为 0 token 的问题。
- 新增 GPT-5.6 Sol、Terra、Luna 的 OpenAI 标准短上下文价格，含 cache read 与 cache write。
- 新增 `gpt-5.3-codex-spark` 价格别名，按公开 `gpt-5.3-codex` 标准价计算。
- 新增 Qwen3.6 27B 的 OpenRouter 公开 API 参考价格：输入 `$0.285/M`、缓存读取 `$0.15/M`、输出 `$2.40/M`。
- 更新 Qwen3.6 Plus 当前国际部署价格：输入 `$0.50/M`、缓存读取 `$0.10/M`、输出 `$3.00/M`。

## 说明

- 本机或自托管 vLLM 的真实成本取决于 GPU、机器和托管费用。`qwen3.6` 默认价格只作为同一 27B 模型的公开 API 成本参考，可在 CCSwitchMulti 定价页按实际部署成本调整。
- `v3.16.5-4` 和 `v3.16.5-5` 的构建流程均已取消，tag 保留作为审计记录；本版使用新的 `v3.16.5-6` tag 发布。
