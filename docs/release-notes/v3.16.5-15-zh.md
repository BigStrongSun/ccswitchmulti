# CCSwitchMulti v3.16.5-15

本版修复 MultiRouter 向导中手动选择 Chat Completions 后，协议可能被后台 Provider 刷新回滚为 Responses 的竞态问题。

## MultiRouter 协议草稿稳定性

- 在“配置核心参数”中将 Qwen、DeepSeek 或其他模型源手动设为 Chat Completions 后，后续“生成路由规则”会稳定保持 `openai_chat`。
- Provider 查询在向导打开期间完成后台刷新时，不再用数据库旧快照覆盖尚未发布的协议选择、模型目录或别名草稿。
- 新增竞态回归测试，模拟手动选择 Chat 后收到旧 `openai_responses` Provider 快照，并验证最终路由仍为 Chat。

## 使用说明

- Chat / Responses 连通性测试只负责探测两个协议入口。两个协议都可用时会保留当前手动选择；只有一个协议可用时才自动切换。
- 用户手动选择的协议优先于探测推荐，不需要反复点击测试按钮。
