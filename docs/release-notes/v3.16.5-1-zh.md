# CCSwitchMulti v3.16.5-1

> [!IMPORTANT]
>
> **此版本尚未适配 OpenAI 新版 ChatGPT 应用。**
> 本版完成的是原版 `cc-switch v3.16.5` 功能跟进与 CCSwitchMulti 兼容合并。Codex Desktop / CLI 的既有接管、路由和模型目录路径仍按已验证逻辑保留；如果新版 ChatGPT 应用改变了桌面端内部协议、模型菜单、渲染器门控或登录态结构，相关能力可能无法正常工作，后续会单独适配。

## 本版定位

`v3.16.5-1` 是 CCSwitchMulti 对原版 `cc-switch v3.16.5` 的跟进版本，重点是把上游原生 Responses 供应商、模型目录、会话分组、供应商预设和平台兼容修复合并回来，同时保留 CCSwitchMulti 自己的 Codex MultiRouter、模型菜单接管、官方 OAuth 保护和发布品牌。

## 主要更新

- 跟进原版 `v3.16.5`：原生 Responses 供应商会生成 Codex 模型目录，MiMo、MiniMax、LongCat、Qwen3-Coder 等已知拒收 `web_search` 的路径会自动禁用该工具。
- 保留 CCSwitchMulti MultiRouter 语义：`modelCatalog` 仍是模型能力和上下文窗口的元数据来源，不等于强制投射到 Codex `/model` 菜单；菜单投射继续由本地路由 / MultiRouter 接管逻辑控制。
- 保留官方 OAuth 边界：ChatGPT 官方登录态不被第三方 API Key 覆盖，official/OAuth route 继续走受管 auth path。
- 合并上游会话管理增强：会话列表支持按供应商和项目目录分组、展开状态持久化、分组批量选择和删除。
- 合并上游供应商预设与资源更新：新增七牛云、FennoAI、ZetaAPI、TeamoRouter、NekoCode、Code0.ai、Amux 等预设和图标资源。
- 修复 native Responses catalog 编辑往返：隐藏能力字段 `supportsParallelToolCalls`、`inputModalities`、`baseInstructions` 会被保留，缺失的 `upstreamModel`、`displayName`、`contextWindow` 不再保存为空字符串。

## 已知限制

- **尚未适配新版 ChatGPT 应用**：如果你已经切到新版 ChatGPT 桌面应用或 OpenAI 后续合并后的新版入口，CCSwitchMulti 的模型菜单解锁、Codex 接管或登录态识别可能需要等待后续专门适配。
- 本版没有改变 Codex Chat Completions 的本地转换边界；Chat 格式供应商仍需要本地路由转换，不能按原生 Responses 直连处理。
- macOS 资产仍取决于仓库是否配置 Apple 签名和公证密钥；未配置时 workflow 会发布未签名 DMG / zip / updater 包。
