# CCSwitchMulti v3.16.5-2

本版是基于 `v3.16.5-1` 的正式修复版，重点补齐新版 Codex / ChatGPT App、多路路由、历史修复和 OpenAI 原生端点透传相关问题。

## 主要修复

- 修复 MultiRouter 下内置 Image Gen 可能被本地代理 404 或误转到默认文本 provider 的问题。
- 新增未知 `/v1/*` OpenAI 原生端点的 raw passthrough：未知端点默认回官方 OpenAI / Codex OAuth 通道，只有明确命中 Qwen、DeepSeek 等第三方 provider 的模型才走第三方链路。
- 修复 `/v1/images/generations` 等图片端点没有注册导致请求还没进入路由层就 404 的问题。
- 修复一键切回 OpenAI 官方后，`~/.codex/config.toml` 里残留本地代理 `base_url` / `wire_api` / provider-owned 字段，导致界面显示官方但请求仍走 CCSwitchMulti 的问题。
- 修复新版 Codex 历史修复入口过于混乱的问题：主流程改为“刷新记录、选择范围、选择目标 provider、确认修复”，并默认跨项目修复全部匹配记录。
- 修复新版历史同步触发时 renderer 注入结果解析错误，导致“修复新版 App 历史”看起来执行但实际没有触发目录同步的问题。
- 修复 active Codex root / SQLite 数据库定位不稳的问题，Standalone 历史修复工具会优先使用当前 active state DB。
- 修复旧版 Codex MultiRouter 内联 OAuth 路由刷新后仍停留在旧模型目录的问题。
- 旧路由现在会按 OAuth 账号安全迁移到对应的 OpenAI Official provider；默认账号优先绑定稳定的 `codex-official`，多账号不会静默串号。
- 官方 OAuth 模型刷新后会同步更新旧路由的精确模型、模型映射与聚合目录，`gpt-5.6-luna`、`gpt-5.6-sol`、`gpt-5.6-terra` 可进入多路路由模型列表。
- 修复 MultiRouter 规则列表里长模型摘要不换行，导致内容覆盖右侧面板的问题。
- 保持新版 Codex `spawn_agent` 保留 schema 兼容，避免子 Agent 工具定义被 CCSwitchMulti 旧扩展字段破坏。

## 使用说明

- 子 Agent 前五候选继续保留用户已有排序；新增 5.6 模型会进入“路由/全部”候选池，不会擅自覆盖用户手工选择。
- 历史修复写入前仍建议完全退出 Codex / ChatGPT App，避免运行中的 app-server 或 WAL 覆盖修复结果。
- 本版会由 GitHub Actions 生成 Windows、macOS、Linux 资产；本地 Windows 测试包也已验证可导出。
