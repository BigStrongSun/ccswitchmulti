# CCSwitchMulti v3.16.5-9

本版修复 Codex 多智能体长会话请求被改坏、Responses-Lite 降级协议错配，以及新建 session 时模型列表偶发为空的问题。

## Codex 多智能体请求

- 修复 official OAuth `/responses` 归一化误删 `agent_message.content` 的问题。
- `message`、`agent_message` 和未来未知 item 类型保守保留原始 `content`；已确认不允许冗余 `content` 的工具调用、工具输出、compaction 等类型继续清理。
- 新增 21 条历史输入的精确回归，覆盖报错中的 `input[20].content`，同时保留明文与加密内容。

## Responses-Lite

- 正常 Lite 转发保留 `additional_tools`、developer message 和原始 input 顺序，不再错误提升成顶层 `tools` / `instructions`。
- Lite 被上游拒绝或命中 fallback 缓存时，会把 Lite body 完整转换成标准 Responses body 后再重试，而不是只删除 header。
- official OAuth 与第三方 native Responses 共用传输边界，但 OAuth 专属字段清理不会泄漏到第三方路径。

## 新 Session 模型列表

- 修复 Codex Desktop `model/list` 首次返回 `{ data: [] }` 时，空结果进入 renderer 查询缓存并导致模型选择入口缺失的问题。
- 兼容层同时覆盖原生 `model/list` 与 `list-models-for-host`，只对已追踪的模型请求回填活动路由目录，避免误改其它 MCP 响应。
- 模型投影读取增加生成 catalog、CCSM models cache、活动 provider 内联 models 的缺失回退，单个目录源短暂不可读时不再为整个 Desktop 会话安装空 payload。
- renderer 兼容层升级到 V5；更新后重新执行“解锁模型菜单”即可安装新版拦截逻辑。
