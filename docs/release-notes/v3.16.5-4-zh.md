# CCSwitchMulti v3.16.5-4

本版修复 Codex MultiRouter 向导、GPT-5.6 Luna 兼容性与新模型用量计费。

## 主要修复

- MultiRouter 设置向导的 Provider 选择改为独立的本次草稿选择。现在可以取消某个 Provider，且不会删除该 Provider 或因页面重新渲染自动重新选中。
- HTTP-only MultiRouter 不再把官方标记为 `prefer_websockets` 的模型写进运行时 Codex 目录。这样不会再把 `gpt-5.6-luna` 暴露给不支持 Responses WebSocket 的本地代理，避免选择后收到上游 `Model not found`。
- Codex 流式 usage 解析兼容 `response.done` 以及事件顶层的 `usage`，修复部分 GPT-5.6 官方 OAuth Responses 流成功返回却被统计为 0 token 的问题。

## 定价更新

- 新增 GPT-5.6 Sol、Terra、Luna 的 OpenAI 标准短上下文价格，含 cache read 与 cache write。
- 新增 `gpt-5.3-codex-spark` 价格别名，按公开 `gpt-5.3-codex` 标准价计算。
- 新增 Qwen3.6 27B 的 OpenRouter 公开 API 参考价格：输入 `$0.285/M`、缓存读取 `$0.15/M`、输出 `$2.40/M`。
- 更新 Qwen3.6 Plus 当前国际部署价格：输入 `$0.50/M`、缓存读取 `$0.10/M`、输出 `$3.00/M`。

## 说明

- 本机或自托管 vLLM 的真实成本取决于 GPU、机器和托管费用，无法由公开 API 单价精确推导。`qwen3.6` 的默认值用于同一 27B 模型的公开 API 成本参考，仍可在 CCSwitchMulti 定价页按实际部署成本调整。
