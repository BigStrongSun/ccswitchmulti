# CCSwitchMulti v3.16.5-7

本版修复 CCSwitchMulti 托管官方 Codex OAuth 请求时的来源身份错误，并恢复 GPT-5.6 Luna 在 HTTP Responses 路径中的正常使用。

## Codex OAuth

- 所有由 CCSwitchMulti 管理并转发到官方 ChatGPT Codex 后端的模型推理请求，现在统一使用官方 Rust CLI 的 `originator: codex_cli_rs`。
- 该规则覆盖普通 Codex Responses、raw passthrough，以及 Claude 入口转换到官方 Codex OAuth 的请求，不再按模型名称做特殊处理。
- 来源头在完整 header 合并和 Provider override 之后强制写入，确保最终只有一个值，避免旧的 `cc-switch`、客户端自带值或重复 header 影响官方模型准入。
- 第三方 Bearer/API Key Provider 不受影响，仍保留其原有请求头语义。
- 官方 OAuth 模型列表请求同步使用相同来源标识；订阅与额度接口保持原有独立客户端身份。

## GPT-5.6 Luna

- 删除把 `prefer_websockets` 误解为“必须使用 WebSocket”的目录过滤。该字段只表示传输偏好，官方 Codex CLI 可以在关闭 WebSocket 时通过 HTTP Responses 使用 Luna。
- Luna、Sol 实测在请求体、OAuth token、账号、endpoint 和其他 headers 相同的情况下，`originator: codex_cli_rs` 返回 HTTP 200；旧的 `originator: cc-switch` 对 Luna 返回 HTTP 404。
- MultiRouter 不再隐藏 Luna，也不会把 Luna 映射成 Terra 或 Sol。实际模型选择继续按用户路由和官方模型目录执行。

## 兼容性

- 本版只改变 CCSwitchMulti 托管的官方 Codex OAuth 模型请求身份，不修改用户保存的 Provider、MultiRouter route、模型映射或本机 Codex 配置。
- `v3.16.5-6` 中关于 Luna 必须依赖 Responses WebSocket 的说明已被本版实测结论取代。
