# CCSwitchMulti v3.16.4-4

v3.16.4-4 是 Codex MultiRouter 配置向导的正式修正版：在保留向导、MultiRouter 工作台、模型源拆分和子 Agent 候选能力的基础上，重点修复首次配置时“点击后状态已变化但用户看不到下一步”的问题，并补齐配置完成后的校验刷新链路。

## 重要修正

### Codex MultiRouter 配置流程可见性

- 在“单独接入模型源”里选择 DeepSeek、GLM、Qwen、OpenAI-compatible 等 Codex 预设后，页面会自动滚到 API Key、Base URL、模型映射和本地接管配置区，避免停在高预设网格上看起来像卡死。
- MultiRouter 工作台没有普通 Codex 模型源时，“没有可选 router”会直接提供“添加模型源”入口，不再只提示用户关闭后自己找入口。
- 创建或编辑路由规则时，行内候选 router 面板会自动滚到可见区域，避免面板展开在视口下方。
- 工具栏进入 Codex 多模型路由时会重置到状态页，不再复用上一次跳转遗留的 provider/tab。
- 工作台内部切换“总览 / 模型源 / 路由规则 / 状态 / 测试发布”时会回到页面顶部，避免从长页面跳转后停在旧滚动位置。

### 配置完成后的校验刷新

- 向导启用 MultiRouter 后，会立即刷新代理监听、Codex 接管、当前 provider、Codex providers 和 usage/logs，不再只等待后台轮询追上。
- 状态页新增“刷新校验”按钮：用户在 Codex 里发送一次请求后，可以立即重新读取链路卡片、最近转发和历史修复触发条件。
- 配置成功判定仍保持严格：必须是当前选中的 MultiRouter 方案有真实 route 成功转发证据，才会进入“配置成功 -> 历史修复”收尾流程。

### 保留已验证的运行时修复

- 保留混合中转 provider 拆分：同一中转同时返回 GPT-like 和非 GPT-like 模型时，可拆成 Responses / Chat 两个独立 provider。
- 保留 Codex Responses-Lite header 兼容修复，避免内部 header 触发第三方或官方上游 400。
- 保留子 Agent 今日流量统计修复，按模型聚合展示子 Agent 数、请求、tokens 和费用。
- 保留子 Agent 派生模型可见性修复，MultiRouter 投影 Codex 模型目录时继续维护 `spawn_agent` 可见候选。
- 保留 Windows 图标与安装链路修复，安装包、便携包、raw exe 和快捷方式继续使用新的 CCSwitchMulti 图标。

## 已知边界

- Linux / macOS 二进制仍需要对应平台或补充工作流构建；Windows 本地构建直接产出 Windows 安装包、便携包、raw exe 和 updater `latest.json`。
- 状态页“刷新校验”只刷新本地状态和日志，不会向上游模型发起新的真实请求；真实转发仍需要用户在 Codex 里发送一次请求。
