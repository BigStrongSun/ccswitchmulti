# CCSwitchMulti v3.16.5-10

本版修复两类启动阻断：升级过上游 v3.17.0 后回到 CCSwitchMulti 无法打开数据库，以及 MultiRouter 异常历史目录导致整个窗口纯白。

## 数据库 v13 兼容

- CCSwitchMulti 现支持 schema v13，可直接打开已被上游 v3.17.0 迁移的 `~/.cc-switch/cc-switch.db`。
- v12 -> v13 迁移会为请求明细和日聚合表补充 `input_token_semantics` 字段，并保留旧数据的未知语义默认值。
- 不再要求用户手动改写 `PRAGMA user_version`；手动降级会造成实际表结构与版本标记不一致，本版已提供正式迁移路径。

## MultiRouter 纯白窗口

- 修复旧 MultiRouter 方案的 `modelCatalog.models` 混入 null、字符串或空模型名时，打开向导直接抛出前端异常的问题。
- 向导只在用户实际打开时挂载，并统一过滤无效目录条目；有效模型会保留，坏条目会被忽略。
- 新增应用根错误边界。后续 React 渲染或提交阶段异常会显示可重新加载的恢复页和错误详情，不再只显示白屏。

## 验证

- MultiRouter 向导、根错误边界和 App 集成相关 Vitest 通过。
- 数据库 v12 -> v13 迁移回归通过。
- TypeScript、Prettier、rustfmt、生产 renderer 构建与 `git diff --check` 通过。
