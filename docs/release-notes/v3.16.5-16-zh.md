# CCSwitchMulti v3.16.5-16

本版本修复 Codex MultiRouter 的两个阻断性故障。

## 修复

- 修复打开多路面板时，在没有已选方案的情况下直接读取空方案配置，导致应用界面以 `null is not an object (evaluating 'e.settingsConfig')` 白屏的问题。
- 修复 MultiRouter 模型目录用通用模板覆盖同 slug 官方模型协议字段的问题。GPT-5.6 Luna、Sol、Terra 现在会保留官方 `use_responses_lite`、`multi_agent_version`、`tool_mode` 等能力，避免 `collaboration.spawn_agent` 因工具结构与后端保留 schema 不一致而被拒绝。
- 保持 `[features.multi_agent_v2] hide_spawn_agent_metadata = true`；子 Agent 模型仍通过 CCSwitchMulti 托管的 custom agent role 选择。

## 验证

- MultiRouter 空方案读取回归测试通过。
- Codex 模型目录 19 项定向测试通过。
- 官方模型 transport、保留工具元数据、models cache 同步及重复同步测试通过。
- Rust 格式检查和 `git diff --check` 通过。
