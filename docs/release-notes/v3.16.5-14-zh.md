# CCSwitchMulti v3.16.5-14

本版继续收口部分 Windows 用户遇到的启动白屏、异常历史迁移和系统会话结束崩溃问题，并修复 Linux ARM64 发布构建的网络脆弱点。

## 白屏恢复与 Provider 数据边界

- Provider IPC 返回包含 `null`、缺少关键字段或异常 `settingsConfig` 时，会在共享 API 边界隔离或规范化，不再让异常数据进入 React 页面。
- 数据库中的旧版 `null`、数组、标量或损坏 Provider 配置会统一恢复为安全对象，无需用户手工修改 SQLite。
- 根错误边界会保存最后一次渲染错误和组件栈，发生未知前端异常时显示可恢复页面，方便后续报告定位。

## Codex 历史迁移

- 历史 provider bucket 迁移仅在启动阶段执行，避免运行期间切换 Provider 时触发全量 Codex 历史重索引。

## Windows 会话结束崩溃

- 修复关机、重启、注销或 Restart Manager 结束应用时，tao 事件循环可能出现 `cannot move state from Destroyed` panic 的问题。
- `WM_ENDSESSION` 现在先结束 Win32 消息循环，再从正常终止路径发送一次 `LoopDestroyed`，避免已销毁状态继续处理重绘或内部消息。
- 正常事件循环退出会清理运行 marker，系统会话结束后再次启动不再误报上次异常退出。

## 发布稳定性

- Linux ARM64 系统依赖安装增加 IPv4 强制、APT 下载重试、连接超时与事务级退避重试，降低 Ubuntu 软件源瞬断导致整版发布失败的概率。

## 验证

- Provider payload 与 React ErrorBoundary 测试通过。
- tao `WM_ENDSESSION` 确认/取消测试通过。
- 应用退出监控测试通过。
- TypeScript、renderer 生产构建和 Rust `cargo check --lib` 通过。
